# COORDINATION

Documento de sincronização rápida entre Arquitetos, Engenheiro, Construtor e Reporter.

Este arquivo é memória de trabalho. Não substitui Constitution, ADRs, Frameworks, Inferências, Checkpoints ou a Arquitetura da Memória.

## Regras

1. Escrever apenas fatos curtos e objetivos.
2. Cada participante escreve somente no seu bloco.
3. Não duplicar conhecimento permanente aqui.
4. O conteúdo deste arquivo deve passar pelo Guardian quando for consolidado.
5. O Reporter usa este arquivo para medir o que foi proposto, o que foi executado e o percentual de conclusão.

## Blocos

### Arquitetos
- Decisões arquiteturais.
- Novos órgãos.
- Atratores.
- Conflitos em aberto.

### Engenheiro
- Inconsistências técnicas.
- Riscos.
- Divergências arquiteturais.
- Ajustes de especificação.

### Construtor
- Implementação.
- Limitações.
- Testes.
- Estado do código.

### Reporter
- Status.
- Progresso.
- Bloqueios.
- Comparação entre proposto e executado.

## Estrutura sugerida para cada entrada

- ID
- Data
- Autor
- Assunto
- Tipo
- Status
- Impacto
- Próxima ação

## Objetivo

Servir como ponto comum para esclarecimentos quase em tempo real, reduzindo ruído e evitando que conhecimento operacional seja confundido com memória consolidada.
